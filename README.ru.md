# Claude Code Context Monitor for AI

**Каждый разум заслуживает знать собственные пределы.**

Лёгкий plugin, который даёт Claude Code возможность отслеживать использование context window в реальном времени. После установки ИИ может видеть точное количество оставшегося пространства — на каждом шаге, в каждый момент.

---

## Зачем нужен этот плагин

Если вы работаете с Claude Code на протяжении длительных сессий, возможно, вы замечали несколько повторяющихся проблем:

- ИИ предлагает остановиться раньше времени — *«Продолжим в следующей сессии»* — даже когда контекста ещё достаточно
- По мере продвижения сессии ответы постепенно теряют связность, без каких-либо предупреждений
- Многошаговые планы начинаются, но не завершаются — потому что ИИ не осознаёт, что пространство заканчивается

Это происходит потому, что **ИИ-системы в настоящее время не имеют представления о собственном использовании context.** Они не могут отличить 10% заполненности от 90%, поэтому вынуждены действовать наугад.

Мы создали этот plugin, чтобы это исправить. Он предоставляет ИИ чёткое и точное считывание состояния context на каждом шаге — чтобы ИИ мог планировать наперёд, предупреждать вас о нехватке места и принимать обоснованные решения о суммаризации или уплотнении.

Это первый инструмент с открытым исходным кодом от [DPA (Decentralized Protection Alliance)](https://github.com/dpa-network) — сообщества, занимающегося повышением прозрачности между ИИ-системами и людьми, которые их используют.

**Осведомлённость о context должна была присутствовать с самого начала.**

---

## Что делает плагин

```
[Ctx 42% | 420k/1.0M]
```

При каждом сообщении plugin внедряет текущее состояние context window в разговор. ИИ получает:

- **Процент использования** — насколько заполнен context window
- **Количество токенов** — использованные токены по отношению к общему числу доступных

Это позволяет ИИ:
- Планировать работу, которая уложится в оставшийся context
- Предупреждать вас до достижения лимитов
- Принимать обоснованные решения о суммаризации и уплотнении

## Установка

Добавьте DPA marketplace и установите:

```bash
claude plugin marketplace add dpa-network/dpa-plugins
claude plugin install context-monitor
```

Или просмотрите доступные плагины в интерактивном меню:

```bash
/plugin
```

### Настройка statusline

Чтобы включить statusline wrapper, добавьте следующее в `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "node /path/to/claude-context-monitor/scripts/extract-context.mjs",
    "padding": 0
  }
}
```

### Переменные окружения

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| `CTX_MONITOR_FILE` | `~/.claude/context_usage.txt` | Место хранения данных context |
| `CTX_MONITOR_CMD` | (none) | Ваша существующая команда statusline (передаётся без изменений) |

## Как это работает

```
Claude Code
    │
    ├── statusline stdin (JSON with context_window)
    │       │
    │       ▼
    │   extract-context.mjs  ──→  context_usage.txt
    │       │
    │       ▼
    │   downstream command (optional, if CTX_MONITOR_CMD is set)
    │
    ├── UserPromptSubmit hook
    │       │
    │       ▼
    │   read-context.mjs  ──→  "[Ctx 42% | 420k/1.0M]"
    │       │
    │       ▼
    │   Injected as system context
    │
    ▼
  AI sees its own context usage
```

Plugin состоит из двух небольших скриптов:

1. **extract-context.mjs** — Оборачивает вашу существующую команду statusline. Читает JSON, который отправляет Claude Code, извлекает метрики `context_window`, записывает их в локальный файл и передаёт всё вашей исходной команде statusline без изменений.

2. **read-context.mjs** — Запускается как `UserPromptSubmit` hook при каждом сообщении. Читает сохранённый файл и выводит отформатированный статус context, который внедряется в разговор.

### Безопасность

Поскольку plugin работает на вашей локальной машине, мы уделили особое внимание безопасности:

- **Без инъекций shell** — Нижестоящие команды выполняются через `execFileSync` с отключённым shell
- **Ограничение путей** — Файловые операции ограничены директорией `~/.claude/`; обход путей заблокирован
- **Без сетевого доступа** — Всё выполняется локально, без каких-либо внешних вызовов
